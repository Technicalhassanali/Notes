What is ADB & AVD?

When we create the Android App.Then main steps is to test the android app that work correctly function or not,that user end not show the UI Problem.The testing is necessary to remove the bugs from the app.The Bugs is problem in the android app.

Types of Error

Two types of Error;

  1. compile time error
  2. Run time error

Compile time Error :A compile-time error generally refers to the errors that correspond to the semantics or syntax.

Run time Error : A runtime error refers to the error that we encounter during the code execution during runtime.

ADB : ADB(Android Debug Bridge) : Android Real Device using USB to install APP and test the App

AVD : AVD(Android Virtual Device) : Android Virtual Device is created with different size and test/debug/check UI of the APP.

What is APK?

  • It is a Package file format used by Android Operating system for distribution and installation of Mobile app
  • It means Android Application Package.
  • Similar to executable file .exe in Windows Operating System.
  • The extension of Apk file is .apk.
  • It is installed on Android Operating System.

It consists of

  1. Application code(.dex files)
  2. Manifest file
  3. Assets
  4. Resource files

Example: It just like a zip(winrar) file that collection of multiple file make and show one file.

How to make Apk File?

In Android Studio the build > Build bundle /Apk option on the topbar is used to built the apk file.

Java(generated)

In this folder the file is used in the run time of the application.All the file check on the run time.

res

res folder is the Resource folder that contain the UI Files.In res the multiple folder are present.

  1. drawable
  2. Layout
  3. mipmap
  4. values
    • themes

Rules of res(Resource) Folder:

When you creating the files in the res(Resource) Folder then you can follow these steps;

  • File Name not starting with Number.
  • Not use Special Character in the File,only underscore(_) is used.
  • File Name not add space,if you use space then you add underscore exept space.

If you not follow the above condition when creating the file in res(Resource) Folder then the not identify the file in runtime.

values

The values folder is contain two files; (1) color.xml (2) strings.xml

color.xml : is used to define the color in xml file.if you have used the color in application 5 time then if we changed to color you not need to change the color from 5 places.Only one place you can go and change the password according to the preference.

two way define color in the color.xml file.

     (1) RGB (3 Color Variation)
     (2) RRGGBB (6 Color Variaton)
     (3) ABRRGGBB (AB is Transparent color)
     R = Red;
     G = Green;
     B = Blue;
If we used 0 then it mean no value and if used FF then mean transparent color. 1 mean color value that we used (0-9) then the character is used.

color are defined in the hexadecimal (0-15) form value.

Examples:

#RGB;                       
#F00; (Red)                 
#0F0; (Green)
________________________________________________________

#FF0000; (Red)

string.xml file is used to store the text that used in the app in multiple time.If we change the text then we don't go to change the text from all over app.Only the string.xml file open then change the text.It would change the text that used most in app. e.g app name,app about us.
